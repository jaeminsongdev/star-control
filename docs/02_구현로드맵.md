# 02. 구현 로드맵

이 문서는 빠른 개요다. 정본 milestone은 `docs/implementation/complete-implementation-roadmap.md`를 따른다.

## M0. 문서와 결정 정렬

- monorepo 디렉터리 구조, schema, contract, policy, registry를 정본화한다.
- runtime stack, fake provider instance, Star Sentinel P0 scope를 결정한다.
- 완전 구현 기본값은 `docs/decisions/0005-full-implementation-defaults.md`로 고정한다.

## M1. Runtime Foundation

- E01 Schema / Runtime Validator
- E02 File-based StateStore
- E03 Artifact Layout Writer
- 목표: canonical examples 검증, `.ai-runs/J-0001` 저장/읽기, path guard, atomic write.

## M2. Provider-neutral Execution

- E04 Provider Registry
- E05 FakeProviderAdapter
- E06 RouterEngine
- E07 ExecutionEngine
- E08 CLI read-only + fake run
- 목표: fake provider로 job 생성, route, workspec, provider output, CLI status/report를 검증한다.

## M3. Validation / Gate

- E09 Star Sentinel P0
- E10 ValidationEngine
- 목표: `AUTO_PASS`, `HUMAN_REVIEW`, `BLOCK` decision과 approval/review pack 흐름을 구현한다.

## M4. v0 Fake E2E

- E11 Integration Smoke
- 목표: fake provider만으로 `route -> execute -> validate -> report` 흐름을 CI에서 반복 가능하게 만든다.

## M5. Local Provider

- local process provider를 먼저 구현한다.
- command policy, sandbox, timeout, cancel, stdout/stderr capture를 검증한다.
- local OpenAI-compatible/local server provider는 공식 문서 refresh 후 붙인다.

## M6. Cloud Provider

- cloud CLI provider를 process/stdio/file handoff 방식으로 붙인다.
- cloud API provider는 `credential_ref`, budget, cost, rate limit, privacy handoff 계약을 먼저 만족해야 한다.

## M7. Daemon / API

- CLI flow가 안정화된 뒤 장시간 queue daemon을 구현한다.
- API는 read-only endpoint부터 시작하고, mutation은 approve/cancel/resume 계약을 따른다.

## M8. UI Shell

- job list, job detail, run timeline, provider output, validation result, approval/review 화면을 제공한다.
- UI는 provider process나 Star Sentinel rule을 직접 실행하지 않는다.

## M9. 운영 안정화와 release readiness

- provider conformance suite, recovery command, retention, observability, audit, cost, redaction을 강화한다.
- release readiness artifact를 만들되, release/deploy/publish 자동화는 별도 승인 전까지 실행하지 않는다.
