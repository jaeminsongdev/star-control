# 사용자 경험과 전체 흐름

## 사용자가 보는 시작점

사용자는 Codex 앱에서 평범한 말로 개발 목표를 입력한다.

    이 프로젝트의 설정 구조를 정리하고 오류 처리를 통일한 뒤 전체 검사를 해줘.

사용자는 별도 Star-Control 화면을 열지 않는다. 설치된 Star-Control Plugin이 개발 작업인지 판단하고 Star-Control 흐름을 시작한다.

단순 설명, 번역, 아이디어 대화처럼 파일이나 외부 상태를 바꾸지 않는 요청은 일반 Codex 대화로 처리할 수 있다. 파일 수정, 명령 실행, 원격 작업처럼 실제 개발 동작을 시작할 때는 Star-Control 실행 기록이 필요하다.

## 전체 흐름

1. 사용자가 Codex 앱에 목표를 입력한다.
2. Star-Control이 목표의 대상, 완료 모습, 제한, 위험을 확인한다.
3. 애매한 내용이 결과를 바꿀 수 있으면 사용자에게 질문한다.
4. 목표를 성격이 같은 실행 단계로 나눈다.
5. 각 단계에 모델, 생각 깊이, 실행 방식, 권한, 검사 방법을 배정한다.
6. 전체 계획과 배정 결과를 사용자에게 보여준다.
7. 사용자가 단계 내용과 순서를 수정할 수 있다.
8. 사용자가 계획을 승인하면 일반적인 실행은 추가 질문 없이 진행한다.
9. Star-Control이 Codex 작업을 만들고 진행 상황을 수집한다.
10. 실패하면 설정된 횟수와 승급 규칙에 따라 다시 시도한다.
11. 필요한 검사만 실행한다.
12. 통과 여부, 변경 내용, 남은 위험, 비용을 기록한다.
13. 다음 단계가 있으면 필요한 내용만 이어서 전달한다.
14. 모든 완료 조건이 충족되면 자동으로 완료 처리한다.

최종 완료에 별도 사람 승인을 강제하지 않는다. 단, 사용자가 설정으로 최종 승인을 요구할 수 있다.

## 질문 원칙

다음 경우에는 먼저 질문한다.

- 같은 문장이 서로 다른 결과를 만들 수 있을 때
- 완료 기준을 알 수 없을 때
- 유료 동작이 필요할 때
- 현재 설정으로 허용되지 않는 동작이 필요할 때
- 여러 안전한 선택 중 사용자의 의도가 결과를 크게 바꿀 때

사소한 구현 세부사항은 계획과 프로젝트 규칙 안에서 스스로 결정한다.

## 계획 화면에 보여줄 내용

- 목표 요약
- 실행 단계와 순서
- 동시에 실행할 수 있는 단계
- 각 단계에 배정된 모델과 생각 깊이
- Max, Ultra, 독립 검토 사용 여부
- 수정 예정 범위
- 실행 예정 검사
- 예상 비용 등급과 유료 동작
- 자동 재시도 횟수
- 완료 조건

## 진행 상황

Codex 앱에는 현재 단계, 진행 중인 작업, 최근 결과, 다음 단계를 보여준다. 터미널에서는 더 자세한 상태와 기록을 확인할 수 있다.

기본 명령은 다음 책임을 가진다.

    star start
    star plan
    star approve
    star status
    star pause
    star resume
    star cancel
    star evidence
    star close

실제 명령 이름과 옵션은 구현 계약 단계에서 확정한다.

## 중단과 이어하기

- 사용자는 작업을 멈추거나 취소할 수 있다.
- 앱을 닫아도 실행 상태와 이어하기 기록은 남는다.
- 재개할 때 전체 대화를 다시 읽지 않는다.
- 실패한 단계는 실패 이유와 이미 시도한 방법을 다음 실행에 전달한다.
- 재시도 횟수와 자동 승급 여부는 설정 파일과 명령어로 바꿀 수 있다.

## 프로젝트별 설정

프로젝트마다 다음 내용을 저장할 수 있다.

- 우선 읽을 문서
- 수정 허용 범위
- 건드리면 안 되는 경로
- 자주 사용하는 검사
- 비용 한도
- 자동 승인 범위
- 병렬 작업 한도
- 원격 저장소 사용 규칙
- 결과 보관 기간
