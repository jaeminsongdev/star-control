# 프로젝트 헌장

## 한 문장 정의

Star-Control은 사용자가 Codex 앱에 입력한 개발 목표를 성격이 같은 실행 단계들로 나누고, 각 단계에 알맞은 Codex 모델·생각 깊이·실행 방식·권한·검사 방법을 배정하여 검증된 결과와 이어하기 기록을 만드는 Windows용 개발 제어 도구다.

## 해결하려는 문제

큰 개발 목표를 Codex에 그대로 맡기면 다음 문제가 생길 수 있다.

- 작업을 지나치게 잘게 나누거나 반대로 너무 크게 처리한다.
- 쉬운 일에도 비싼 모델을 사용한다.
- 중요한 일에 생각 깊이가 부족할 수 있다.
- 완료했다고 보고했지만 필요한 검사가 빠질 수 있다.
- 긴 작업이 끊겼을 때 다음 작업이 같은 조사를 반복한다.
- 동시에 진행한 변경이 서로 충돌할 수 있다.

Star-Control은 이 문제를 계획, 배정, 실행 통제, 검사, 증거, 이어하기로 해결한다.

## 작업을 나누는 기준

Star-Control은 파일 하나나 함수 하나 단위로 일을 쪼개지 않는다. 다음 중 하나가 달라질 때 새 단계로 나눈다.

- 필요한 모델이나 생각 깊이
- 실행 방식
- 사용 권한
- 검사 방법
- 먼저 끝나야 하는 다른 단계
- 동시에 진행해도 되는지 여부
- 실패했을 때 되돌릴 경계

성격이 같다면 파일이 많고 작업량이 커도 한 단계로 묶을 수 있다.

## 대상 사용자

- 처음에는 개인 개발자가 주요 사용자다.
- 최종 결과는 공개 배포할 수 있어야 한다.
- 공개 사용자는 설정을 바꾸지 않아도 안전하게 시작할 수 있어야 한다.
- 개발자는 필요하면 더 자동화된 개인 설정을 선택할 수 있어야 한다.

## 지원 환경

- Windows만 지원한다.
- Codex 앱을 사용자 입력 화면으로 사용한다.
- 터미널을 상태 확인과 직접 조작 화면으로 사용한다.
- Codex App Server를 실제 Codex 작업 생성과 제어 통로로 사용한다.

## 동등하게 중요한 목표

품질, 안전, 비용, 속도, 사용 편의는 하나를 버리고 하나를 얻는 순위가 아니다.

1. 요구된 품질과 안전 기준을 먼저 만족한다.
2. 그 기준을 만족하는 방법 중 예상 총비용과 총시간이 적은 방법을 고른다.
3. 실패와 재작업까지 포함한 실제 결과로 선택 규칙을 개선한다.

## 최종 제품 범위

- 자연어 목표 입력
- 질문을 통한 목표 명확화
- 단계 분해와 순서 결정
- 모델·생각 깊이·실행 방식 자동 배정
- 사용자의 배정 결과 수정
- Codex 작업 자동 생성과 진행 제어
- 병렬 실행과 작업 복사본 관리
- 변경 범위 감시
- 필요한 검사 자동 선택
- 완료 증거와 이어하기 기록
- 비용·시간·실패·재작업 측정
- 여러 프로젝트에 걸친 작업
- 인터넷 조사
- 로컬 변경 기록, 원격 업로드, 검토 요청, 병합
- 공개 배포 가능한 Plugin·MCP·Hook·CLI 묶음

## 만들지 않는 것

- 자체 AI 모델
- 로컬 AI 연결
- Claude, Gemini 등 다른 AI 제공자 연결
- OpenAI API 직접 호출
- 여러 AI 제공자 사이의 전환과 장애 대체
- GPU 관리
- OpenAI 호환 서버 연결
- 브라우저 기반 Star-Control 화면
- 자체 예약 실행 기능
- 범용 개발 환경이나 운영체제 관리 도구

## 구현 원칙

- Codex가 이미 제공하는 기능은 다시 만들지 않고 연결해서 사용한다.
- 실행 결과를 믿는 대신 필요한 검사를 수행한다.
- 검사 비용이 효용보다 크면 실행하지 않는다.
- 기록은 다음 판단에 필요한 만큼만 남긴다.
- 사용자 설정은 Star-Control이 Codex 자체의 더 강한 제한을 약화시키지 못한다.
- 최종 범위는 축소하지 않되 구현은 검증 가능한 단계로 나눈다.
