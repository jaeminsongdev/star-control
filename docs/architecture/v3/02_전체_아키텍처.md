# 02. 전체 아키텍처

## 1. 최상위 구조

```text
User
  ↓
Star-Control CLI / UI
  ↓
Router Engine
  ↓
Core Specs + Policies + Capabilities
  ↓
Provider Adapter
  ├─ CodexAdapter
  ├─ ClaudeAdapter
  ├─ GeminiAdapter
  ├─ CursorAdapter
  ├─ CopilotAdapter
  ├─ JulesAdapter
  ├─ DevinAdapter
  └─ LocalModelAdapter
  ↓
Worker Runs
  ↓
Reports / Validation / Review / Final Output
```

## 2. 주요 컴포넌트

### 2.1 Router Engine

역할:

- 사용자 요청을 Job으로 등록한다.
- Router worker를 실행하거나 내부 규칙으로 route를 만든다.
- 작업 규모와 위험도를 분류한다.
- stage 목록을 만든다.
- stage별 worker/provider를 배정한다.
- WorkSpec을 생성한다.
- RunState를 업데이트한다.

### 2.2 Provider Adapter Engine

역할:

- 공통 WorkSpec을 provider별 실행 명령으로 변환한다.
- Codex면 `codex exec --profile ...`을 실행한다.
- Local Ollama면 `ollama run ...` 또는 API 호출을 실행한다.
- 결과를 공통 ReportSpec으로 변환한다.

### 2.3 Policy Engine

역할:

- 위험도 판단
- 승인 필요 여부 판단
- provider 선택 제한
- command-policy 적용
- budget/cost 제한
- scope guard 적용

### 2.4 Run Engine

역할:

- `runs/J-0001/` 생성
- stage 상태 전이
- worker 실행
- output/report 수집
- 재시도/중단/차단 처리

### 2.5 Hook Engine

역할:

- before-route
- after-route
- before-worker
- after-worker
- after-validation
- after-review
- on-blocked
- on-done

등 이벤트에 결정론적 작업을 붙인다.

### 2.6 Validation / Review Engine

역할:

- 프로젝트 검증 명령 탐색
- 빌드/테스트/린트/타입체크 실행
- 실패 로그 요약
- 리뷰 worker 실행
- 최종 판정 생성

## 4. Provider별 native 기능 활용 전략

Provider가 native 기능을 제공하면 사용한다. 하지만 원본 상태와 정책은 Star-Control이 보유한다.

예:

```text
Star-Control Goal Engine
  ├─ Codex: native /goal 사용 가능
  ├─ Claude: /loop skill 또는 agent loop로 emulation
  ├─ Gemini: 자체 goal state로 emulation
  ├─ Cursor: background agent + validation loop로 emulation
  └─ Local: Router Engine loop로 emulation
```

## 5. Codex 1차 Adapter의 위치

Codex는 1차 구현 provider다.

Codex용 산출물:

```text
%USERPROFILE%\.codex\low-router.config.toml
%USERPROFILE%\.codex\worker-impl.config.toml
%USERPROFILE%\.codex\worker-review.config.toml
%USERPROFILE%\.codex
ules\command_block.rules
%USERPROFILE%\.agents\skills\...
```

하지만 이 파일들은 원본이 아니라 `renderers/codex/`가 생성하거나 유지하는 provider-specific 파일이다.

## 6. 디렉터리 책임

```text
schemas/       = 공통 데이터 계약
roles/         = worker 역할 지침
skills/        = 반복 절차
policies/      = 판단/권한/위험 규칙
hooks/         = lifecycle 자동화
providers/     = provider 실행 정의
renderers/     = provider-specific 설정 생성기
runs/          = 실행 상태와 산출물
quality/       = 검증/리뷰/증거 엔진
retrieval/     = 코드/문서 검색
vcs/           = git/worktree/diff/PR
control-plane/ = UI와 관제
```
