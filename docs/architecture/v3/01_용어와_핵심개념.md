# 01. 용어와 핵심 개념

## 1. 핵심 용어

| 용어 | 의미 |
|---|---|
| Job | 사용자 요청 하나에서 시작되는 전체 작업 단위 |
| Run | 특정 Job 안에서 실행되는 하나의 처리 흐름 |
| Thread | Provider 내부의 대화/세션/실행 컨텍스트 |
| Worker | 특정 역할을 수행하는 AI 또는 도구 실행 단위 |
| Provider | Codex, Claude, Gemini, Cursor, Local Ollama 같은 실행 백엔드 |
| Adapter | Provider를 Star-Control 공통 인터페이스에 맞추는 실행 계층 |
| Renderer | Star-Control 공통 설정을 provider-specific 설정 파일로 변환하는 계층 |
| RoleSpec | Worker의 역할 지침 |
| WorkSpec | Worker에게 전달되는 구체적 작업 명세 |
| ReportSpec | Worker가 반환해야 하는 표준 보고 형식 |
| RunState | Job/Run/Thread/Worker의 현재 상태 저장소 |
| Capability | Provider 또는 Star-Control이 제공하는 기능 단위 |
| Policy | 위험도, 권한, 승인, 모델 선택, 명령 차단 규칙 |
| Hook | lifecycle event 전후에 실행되는 결정론적 작업 |
| Skill | 반복 절차/워크플로우를 재사용하기 위한 지침 패키지 |

## 2. Provider별 용어 매핑

| Star-Control | Codex | Claude Code | Gemini CLI | Cursor | GitHub Copilot |
|---|---|---|---|---|---|
| Context File | AGENTS.md | CLAUDE.md | GEMINI.md | Rules | copilot-instructions.md |
| Skill | Skill | Skill/Command | Skill | Skill | Agent Skill |
| Worker | Subagent / exec profile | Subagent / Agent Team member | Subagent | Agent / Background Agent | Custom Agent / Cloud Agent |
| Thread | Thread / Session | Session | Session | Conversation | Agent session |
| Goal | /goal | /loop 또는 plan+loop | AI Router emulation | AI Router emulation | Cloud task / issue task |
| Plan | /plan | /plan | Plan Mode | Plan Mode | implementation plan |
| Rule | `.rules` | permissions/settings/rules | safety policy/settings | rules/permissions | instructions/policies |
| Hook | hooks | hooks | hooks | provider-specific | hooks/automation where available |
| Extension | plugin | plugin | extension | marketplace/rules pack | skills/custom agents/MCP |

## 3. 자체 용어를 써야 하는 이유

Provider마다 같은 개념을 다른 이름으로 부른다. 예를 들어 Codex의 subagent, Claude의 subagent, Gemini의 subagent, Cursor의 background agent, GitHub Copilot cloud agent는 모두 “위임된 작업자”라는 공통 개념을 갖지만 lifecycle과 권한 모델은 다르다.

Star-Control은 내부적으로 provider-neutral 용어를 사용하고, provider adapter에서만 해당 도구의 용어로 변환한다.

## 4. Thread와 Run의 차이

- **Thread**: provider 내부 대화/세션 컨텍스트. 예: Codex thread id, Claude session id.
- **Run**: Star-Control이 관리하는 실행 단위. 하나의 run이 하나 이상의 provider thread를 사용할 수 있다.

예:

```text
J-0001: 스톱워치 만들기
  run: route
    thread: codex low-router exec session
  run: implement
    thread: codex worker-impl exec session
  run: review
    thread: codex worker-review exec session
```

## 5. Skill과 Hook의 차이

- Skill은 LLM이 읽고 따르는 절차 지침이다.
- Hook은 LLM 판단 없이 실행되는 결정론적 단계다.

예:

- `code-review.md`: 어떤 항목을 리뷰할지 알려주는 Skill.
- `after-worker.yaml`: worker 완료 후 changed_files를 수집하는 Hook.

## 6. Plan과 Goal의 차이

- Plan은 구현 전 설계·검토·승인 단계다.
- Goal은 완료 조건이 있는 목표를 반복 추진하는 실행 상태다.

예:

```text
Plan: 스톱워치 구현을 위해 어떤 파일을 고칠지 계획한다.
Goal: 스톱워치 기능이 빌드/테스트/스모크 테스트를 통과할 때까지 반복한다.
```
