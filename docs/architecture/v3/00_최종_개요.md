# 00. Star-Control 최종 개요

## 1. 프로젝트 정의

**Star-Control**은 여러 클라우드 AI와 로컬 모델을 공통 작업 규격으로 제어하는 멀티 AI 작업 오케스트레이터다.

목표는 다음이다.

- 사용자가 자연어로 작업을 요청한다.
- Star-Control이 작업을 분해한다.
- 적절한 provider/model/worker/thread를 선택한다.
- 새 실행 쓰레드 또는 worker run을 만든다.
- 구현, 검증, 리뷰, 폴리싱, 보고를 표준화한다.
- Codex뿐 아니라 Claude Code, Gemini CLI, Cursor, GitHub Copilot, Jules, Devin, 로컬 모델을 같은 방식으로 연결한다.

## 2. 가장 중요한 설계 전환

처음에는 Codex 기준으로 `low-router.config.toml`, Codex Skills, Codex Rules, Codex Subagents를 설계했지만, 최종 방향에서는 이것들이 모두 **Provider-specific adapter 산출물**로 내려간다.

| 이전 관점 | 최종 관점 |
|---|---|
| Codex low-router profile이 중심 | Star-Control Router Engine이 중심 |
| Codex Skill이 절차 원본 | Star-Control Skill/Procedure가 원본 |
| Codex `.rules`가 안전정책 원본 | `policies/command-policy.yaml`이 원본 |
| Codex subagent가 worker 원본 | Star-Control Worker Role이 원본 |
| Codex exec/resume가 thread 원본 | Star-Control Run/Thread Engine이 원본 |
| Codex 설정을 직접 관리 | Provider Renderer가 Codex 설정을 생성 |

## 3. 핵심 원칙

### 3.1 Provider-neutral 원본 우선

모든 역할, 정책, 스키마, 워크플로우는 Star-Control 내부의 공통 파일로 먼저 정의한다.

```text
roles/*.md
policies/*.yaml
schemas/*.json
skills/*.md
hooks/*.yaml
```

Codex/Claude/Gemini/Cursor/GitHub용 설정은 이 공통 원본에서 생성되는 adapter 산출물이다.

### 3.2 AI는 판단자, Router Engine은 실행 제어자

AI에게 프로세스 생성, 세션 복구, 실패 재시도, 상태 업데이트를 전부 맡기지 않는다.

```text
AI Worker: 설계/구현/검토/요약/판단
Router Engine: 상태 관리/실행 제어/정책 적용/재시도/중단/보고 수집
```

### 3.3 로컬 모델은 보조 worker

로컬 모델은 비용 절감과 반복 작업 보조에 사용한다.

좋은 용도:

- 파일 요약
- 구현 초안
- 테스트 후보
- 로그 요약
- diff 1차 리뷰
- 단순 반복 변환

금지/비추천:

- 보안/권한 최종 판단
- DB/스키마 최종 설계
- 동시성/파일시스템 핵심 변경
- 공개 API 변경
- 대형 리팩토링 최종 구현
- 테스트 실패 우회 판단

### 3.4 Provider native 기능은 쓰되, 종속되지 않는다

Codex의 Goals, Skills, Hooks, Rules, Subagents, CLI exec/resume/fork는 강력한 기능이지만 Star-Control의 원본 기능은 아니다. Codex Skills는 반복 workflow 패키지이고, Codex CLI profile은 named config layer, `.rules`는 Starlark 기반 execpolicy, Goals는 durable objective라는 점을 adapter 설계에 반영한다. 출처: https://developers.openai.com/codex/skills, https://developers.openai.com/codex/config-advanced, https://developers.openai.com/codex/rules, https://developers.openai.com/codex/use-cases/follow-goals.

## 4. 최종 시스템 구성

```text
Star-Control
  ├─ Core Spec
  ├─ Capability Registry
  ├─ Provider Feature Matrix
  ├─ Router Engine
  ├─ Plan Engine
  ├─ Goal Engine
  ├─ Skill / Procedure Engine
  ├─ Command Engine
  ├─ Hook Engine
  ├─ Rule / Permission Engine
  ├─ Worker Engine
  ├─ Thread / Run Engine
  ├─ Context / Memory Engine
  ├─ Tool / MCP Engine
  ├─ Retrieval Engine
  ├─ VCS / Worktree Engine
  ├─ Validation Engine
  ├─ Review / Critic Engine
  ├─ Budget / Cost Engine
  ├─ Extension Package Engine
  └─ Control Plane UI
```
