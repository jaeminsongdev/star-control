# 05. 구현 로드맵

이 문서는 Star-Control을 실제로 구현하는 순서를 정의한다.

## 전체 우선순위

```text
1. Core Spec
2. Registry / Policy
3. Role / Skill 원본
4. Codex Adapter MVP
5. Run Engine
6. Validation / Review Engine
7. Hook / Policy Engine
8. Local Model Adapter
9. Plan / Goal Engine
10. Multi Provider
11. Extension Package Engine
12. Control Plane UI
```

## Phase 0. 설계 고정

### 만들 것

```text
README.md
router.yaml
docs/architecture.md
docs/terms.md
```

### 확정할 것

- job, run, thread, worker, provider, capability, policy, workspec, report 용어
- provider-neutral 원본 우선 원칙
- AI는 판단자, Router Engine은 실행 제어자 원칙
- provider-specific 설정은 renderer 산출물이라는 원칙

### 완료 기준

- 시스템 정의를 1문장으로 설명 가능
- 각 폴더의 책임이 명확함
- Codex 전용 파일과 Star-Control 원본 파일이 분리됨

- 위험 작업은 approval_required로 표시 가능

## Phase 5. Run Engine MVP

### 만들 것

```text
core/run_manager
core/state_store
core/artifact_store
core/session_registry
```

### 기능

- job_id 발급
- run directory 생성
- stage 상태 전이
- worker 실행
- report 수집
- 재시도/중단

### 완료 기준

```powershell
star-control run "스톱워치 만들어줘" --project "D:\개발\프로젝트A"
```

실행 시 다음이 생성되어야 함.

```text
runs/J-0001/request.md
runs/J-0001/route.json
runs/J-0001/workspec-impl.md
runs/J-0001/implement/output.md
runs/J-0001/implement/report.json
runs/J-0001/review/output.md
runs/J-0001/review/report.json
runs/J-0001/final-report.md
```

## Phase 6. Validation / Review Engine

### 만들 것

```text
quality/validation-engine
quality/review-engine
quality/evidence-engine
```

### 기능

- 프로젝트 검증 명령 탐색
- 관련 테스트 추천
- 실패 로그 요약
- 리뷰 worker 실행
- 최종 판정

### 완료 기준

- 검증 성공/실패가 report.json에 표준화됨
- 테스트 미실행은 통과로 표시되지 않음
- 리뷰에서 BLOCK이면 다음 단계 중단

## Phase 9. Plan / Goal Engine

### Plan Engine

- 구현 전 조사
- 계획 작성
- 승인 전 파일 수정 금지
- 계획 승인 후 implement stage 전환

### Goal Engine

- 목표 등록
- 반복 실행
- evidence 기반 완료 판단
- pause/resume/clear
- iteration budget 제한
