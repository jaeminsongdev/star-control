# Custom Dev Verification Platform v4 Curated

작성일: 2026-06-28

이 문서 세트는 v3의 “모든 기능을 다 담는 백과사전식 설계”를 다시 검토해서, 사용자의 실제 개발 상황에 맞는 **장기적으로 구현할 가치가 있는 커스텀 도구/검증기만 선별**한 v4 설계다.

## 운영 전제

- 주 구현자: 클라우드 AI
- 사용자: 1인 개발자, 최종 검토/승인 담당
- 도구 전략: CodeQL, Semgrep, Sonar, Snyk, Trivy 같은 범용 도구를 붙이는 것이 아니라, 그들이 제공하는 개념 중 필요한 것만 자체 커스텀 도구로 구현
- 대상 프로젝트: Rust/CLI/GUI/AI 관제/RAG/개발도구 계열로 확장 가능
- 핵심 목적: “기능이 많은 정적분석기”가 아니라, **AI가 만든 변경을 사람이 짧은 시간 안에 승인/반려할 수 있게 만드는 검증 플랫폼**

## v3에서 수정한 방향

v3는 누락 방지를 위해 405개 기능까지 확장했다. v4는 이를 다시 선별해 **110개 핵심/조건부 기능**으로 정리했다.

- 유지: 작업 통제, 테스트 무력화 방지, 정책/진단/원장, 리뷰팩, 아키텍처/계약, 보안/시크릿, 의존성/라이선스, 결정성, 검증기 자기보호
- 축소: full multi-language type checker, full CFG/DFG/taint, full formal verification, full TUF/in-toto/SLSA, 전체 GUI/a11y/e2e, 전면 privacy flow
- 조건부: GUI/Web 검증, AI/RAG prompt 검증, DB migration, release provenance, fuzz/mutation, compatibility matrix

## 핵심 결론

> 네 상황에서는 “모든 검증 분야를 최고 수준으로 구현”하는 게 아니라, **AI 주 구현자 체계에서 실제 사고를 줄이고 최종검토 비용을 줄이는 검증기**를 먼저 구현해야 한다.

따라서 v4의 중심은 아래 7개다.

```text
1. repo/diff/policy/diagnostic/ledger 기반
2. sentinel task / agent capability / AI claim 검증
3. test weakening / validator self-guard / regression corpus
4. hardcode/fake implementation / hallucinated API 탐지
5. architecture boundary / public API / schema 계약 검증
6. secret/security-lite/dependency/license/SBOM-lite
7. review pack / approval gate / rejection prompt
```
