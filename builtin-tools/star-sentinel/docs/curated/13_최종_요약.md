# 13. 최종 요약

## 최종 판단

v4에서는 v3의 405개 기능을 무작정 유지하지 않고, 실제 사용자 상황에 맞는 **75개 장기 구현 가치 기능**만 남겼다.

## 가장 먼저 만들 것

```text
1. repo map / file classifier / diff parser
2. policy registry / diagnostic model / ledger
3. sentinel task freezer / scope validator
4. test weakening detector
5. secret detector
6. dependency approval validator
7. validator selfguard
8. AI claim verifier
9. review pack generator
10. approval gate
```

## 장기적으로 꼭 가져갈 것

```text
- architecture boundary
- public API/config/schema contract
- SBOM-lite / license / OSV matcher
- determinism validator
- regression/adversarial corpus
- sandbox validation runner
- docs/ADR/AGENTS consistency
- release integrity subset
- AI/RAG prompt/tool validation subset
```

## 무리해서 만들지 말 것

```text
- full type checker
- full taint engine
- full formal verification
- full TUF/in-toto/SLSA
- 전체 GUI/e2e/a11y/visual regression
- 전체 compatibility matrix
```

## 한 줄 결론

> 네 상황에 필요한 것은 “거대한 정적분석 백과사전”이 아니라, **AI가 구현한 변경을 자동으로 심사하고, 장기적으로 코드 품질·보안·계약·테스트 신뢰성을 지켜주는 선별형 커스텀 검증 플랫폼**이다.
