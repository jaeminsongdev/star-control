# 04. 전체 아키텍처 v4

## 1. 최종 구조

```text
star-sentinel
  ├─ repo              # repo map, file classifier, diff parser, risk path
  ├─ policy            # policy registry, rule DSL, allowlist, approval policy
  ├─ analyze           # language adapters, symbol index, dependency graph
  ├─ validate          # validators grouped by profile
  ├─ evidence          # ledger, logs hash, artifacts, diagnostics
  ├─ corpus            # regression/adversarial/golden fixtures
  ├─ runner            # sandboxed validation runner
  ├─ review            # review pack, risk summary, rejection prompt
  ├─ gate              # AUTO_PASS / HUMAN_REVIEW / BLOCK
  └─ selfguard         # validator/policy/CI self-protection
```

## 2. 데이터 흐름

```text
Star Sentinel Task
  ↓
AI Implementation
  ↓
Git Diff + Validation Logs
  ↓
Repo/Diff Analyzer
  ↓
Policy + Validators
  ↓
Diagnostics + Ledger
  ↓
Review Pack
  ↓
Approval Gate
  ↓
사용자 최종 승인/반려
```

## 3. 핵심 설계 원칙

### 3-1. 분석과 판정 분리

- validator는 진단을 만든다.
- gate는 진단과 정책을 합쳐 승인/반려를 결정한다.
- review pack은 사람이 읽을 요약만 만든다.

### 3-2. 모든 결과는 증거 기반

- AI 보고는 신뢰하지 않는다.
- diff, log, command exit code, artifact hash, validator diagnostic만 신뢰한다.
- 검증하지 않은 항목은 “미검증”으로 표시한다.

### 3-3. 외부 표준은 출력/모델 참고용

- 내부 diagnostic은 SARIF에 매핑 가능하게 설계한다.
- dependency identity는 PURL/SPDX/CycloneDX와 호환되게 설계한다.
- release subset은 SLSA/in-toto/TUF의 작은 subset만 쓴다.

## 4. 최소 모듈 경계

```text
repo는 policy를 모른다.
policy는 source code parser를 모른다.
validator는 gate 판정을 직접 내리지 않는다.
gate는 source code를 직접 읽지 않고 diagnostics와 policy만 본다.
review는 사람이 읽을 수 있는 summary만 만든다.
selfguard는 validator/policy/CI 변경을 별도로 감시한다.
```

## 5. 프로파일

```text
quick    : diff/scope/secret/test weakening/AI claim/review pack
near     : quick + near tests + architecture-lite + dependency diff
full     : near + all P1/P2 validators
security : secret + security-lite + dependency/license + auth risk
release  : full + SBOM-lite + artifact hash + release manifest
validator: validator self-test + corpus + golden diagnostics
```
