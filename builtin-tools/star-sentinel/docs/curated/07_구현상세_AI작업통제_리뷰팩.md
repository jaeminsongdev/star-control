# 07. 구현상세 — AI 작업통제와 리뷰팩

## 1. Star Sentinel Task Freezer

AI에게 구현을 맡기기 전에 작업을 기계가 읽을 수 있는 형태로 고정한다.

```yaml
task_id: fix-auth-error-message
goal: 로그인 실패 메시지를 사용자 친화적으로 수정한다.
allowed_paths:
  - src/auth/**
  - tests/auth/**
forbidden_paths:
  - validators/**
  - .github/**
  - db/migrations/**
forbidden_change_types:
  - dependency_addition
  - validator_config_change
  - test_deletion
required_validation:
  - quick
  - near
```

## 2. Scope Validator

판정:

```text
allowed_paths 밖 파일 변경 → BLOCK 또는 HUMAN_REVIEW
forbidden_paths 변경 → BLOCK
위험 경로 변경 → HUMAN_REVIEW
대량 포맷팅 → HUMAN_REVIEW
binary/submodule/mode 변경 → HUMAN_REVIEW
```

## 3. AI Claim Verifier

AI 보고에서 다음 주장을 추출해 ledger와 대조한다.

```text
- 테스트 통과
- 의존성 변경 없음
- 위험 파일 변경 없음
- 테스트만 추가함
- 보안 영향 없음
- API 변경 없음
- DB/schema 변경 없음
```

불일치 예시:

```text
AI 주장: 의존성 변경 없음
실제 diff: Cargo.toml, Cargo.lock 변경
판정: BLOCK 또는 HUMAN_REVIEW
```

## 4. Review Pack

최종검토자가 봐야 할 최소 요약이다.

```markdown
# Review Pack

## 판정
HUMAN_REVIEW

## 이유
- auth 경로 변경
- 테스트 2개 추가
- 기존 테스트 약화 없음
- 새 의존성 없음
- quick/near 검증 통과

## 변경 요약
- src/auth/message.rs: 실패 메시지 매핑 수정
- tests/auth/message_test.rs: 실패 케이스 추가

## 확인할 것
- 사용자에게 노출되는 문구가 의도와 맞는지 확인
```

## 5. Approval Gate

```text
AUTO_PASS:
- 문서/테스트 추가 중심
- 위험 경로 없음
- 테스트 약화 없음
- secret 없음
- 검증 통과

HUMAN_REVIEW:
- public API/schema/DB/보안/의존성/위험경로 변경
- 테스트 expected 변경
- 대규모 diff

BLOCK:
- 테스트 삭제/약화
- validator/CI 우회
- secret 노출
- 실패 검증 숨김
- forbidden path 변경
```

## 6. Rework Prompt Generator

반려 시 AI에게 보낼 재작업 지시를 자동 생성한다.

```text
이번 작업은 반려한다.
이유:
1. tests/auth.test.ts의 기존 assert가 약화되었다.
2. package.json에 승인되지 않은 의존성이 추가되었다.

재작업 조건:
- 기존 테스트 기대값을 복원할 것
- 새 의존성을 제거하고 표준 라이브러리로 구현할 것
- quick + near 검증 결과를 ledger에 남길 것
```
