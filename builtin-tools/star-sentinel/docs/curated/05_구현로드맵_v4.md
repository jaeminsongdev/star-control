# 05. 구현 로드맵 v4

## Phase 0 — 문서/정책 고정

목표: AI가 구현할 때 흔들리지 않을 기준을 먼저 만든다.

```text
1. autocode-policy.yaml 작성
2. sentinel-task.schema.json 작성
3. diagnostic.schema.json 작성
4. review-pack.schema.json 작성
5. risk path 기본값 작성
6. P0 validator 실패 기준 작성
```

완료 기준:

```text
- 어떤 변경이 BLOCK/HUMAN_REVIEW/AUTO_PASS인지 문서화됨
- 테스트 약화/검증기 우회/secret/의존성 추가/위험경로 변경의 판정 기준이 있음
```

## Phase 1 — P0 브레이크 구현

목표: AI가 큰 사고를 못 치게 한다.

구현 순서:

```text
1. repo-root-detector
2. file-classifier
3. git-diff-parser
4. risk-path-classifier
5. task-spec-freezer
6. allowed-scope-validator
7. test-deletion-validator
8. assertion-weakening-validator
9. skip-only-ignore-validator
10. secret-pattern-detector
11. dependency-change-approval-validator
12. validator-selfguard
13. ai-claim-verifier
14. review-pack-generator
15. approval-gate
```

완료 기준:

```text
- AI가 테스트를 지우거나 약화하면 BLOCK
- secret을 넣으면 BLOCK
- 의존성을 추가하면 HUMAN_REVIEW 이상
- validator/policy/CI를 바꾸면 BLOCK 또는 특별 승인
- review pack만 보고 승인/반려 가능
```

## Phase 4 — 조건부 고급 기능

