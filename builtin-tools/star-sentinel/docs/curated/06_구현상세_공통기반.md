# 06. 구현상세 — 공통 기반

## 1. Repo Map

### 입력

```text
project_root
policy.yaml
optional sentinel_task.yaml
```

### 출력

```json
{
  "root": "D:/dev/project",
  "languages": ["rust", "typescript"],
  "manifests": ["Cargo.toml"],
  "lockfiles": ["Cargo.lock"],
  "risk_paths": [".github/**", "validators/**", "db/migrations/**"],
  "source_files": [],
  "test_files": [],
  "docs_files": [],
  "generated_files": [],
  "vendor_files": []
}
```

### 구현

- root marker 기반 탐지: `.git`, `AGENTS.md`, `autocode.yaml`, 언어별 manifest
- 파일 분류는 확장자보다 경로/manifest 신호를 우선
- generated/vendor/output은 policy로 추가 정의 가능

## 2. Diff Model

### 필드

```text
file_path
change_type: added|modified|deleted|renamed|mode_changed|binary
old_path
new_path
added_lines
removed_lines
hunks
is_test
is_source
is_risk_path
is_generated
```

### 구현 포인트

- binary/submodule/file mode 변경은 항상 HUMAN_REVIEW
- generated/vendor 변경은 기본 BLOCK
- risk path 변경은 최소 HUMAN_REVIEW
- test file 변경은 test weakening validator로 추가 분석

## 3. Diagnostic Model

SARIF에 매핑 가능하도록 설계한다.

```json
{
  "rule_id": "test.assertion_weakened",
  "severity": "block",
  "confidence": 0.92,
  "message": "기존 assert가 더 약한 조건으로 변경됨",
  "location": {"file": "tests/auth.test.ts", "line": 42},
  "evidence": ["old: expect(status).toBe(403)", "new: expect(status).not.toBe(500)"],
  "remediation": "테스트 기대값을 원래 계약 기준으로 복원하거나 별도 승인 필요",
  "fingerprint": "sha256:..."
}
```

## 4. Policy Model

```yaml
risk_paths:
  - ".github/**"
  - "validators/**"
  - "db/migrations/**"
  - "auth/**"
  - "security/**"

gates:
  block:
    - test_deleted
    - assertion_weakened
    - secret_detected
    - validator_disabled
  human_review:
    - new_dependency
    - public_api_changed
    - db_migration_changed
    - risk_path_changed
```

## 5. Ledger

Ledger는 append-only에 가깝게 운용한다.

```text
runs/<task_id>/
  sentinel_task.yaml
  repo_map.json
  diff.json
  diagnostics.json
  validation_runs.json
  review_pack.md
  approval.json
  logs/
  artifacts/
```

완료 보고는 ledger에서 생성해야 하며, AI가 수동으로 작성한 “검증 통과” 문구를 그대로 신뢰하지 않는다.
