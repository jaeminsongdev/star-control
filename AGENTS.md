# Star-Control 작업 지침

## 기본 원칙

- 응답과 새 문서는 한국어를 우선한다.
- 설정 이름, 명령어, 코드 이름, 파일명은 필요한 경우 원문을 유지한다.
- 현재 설계 기준은 docs/ 아래 새 문서뿐이다.
- legacy/는 로컬 읽기 전용 참고자료이며 수정하거나 현재 기준으로 인용하지 않는다.
- 새 문서만 읽어도 설계 전체를 이해할 수 있어야 한다.

## 현재 단계

- 현재는 문서 설계 단계다.
- 문서 설계가 확정되기 전에는 제품 코드를 작성하지 않는다.
- 작업은 지나치게 작은 조각이 아니라 모델·생각 깊이·실행 방식이 달라지는 단계 단위로 나눈다.
- 로컬 AI, 다른 AI 제공자, OpenAI API 직접 호출, 브라우저 UI, 자체 예약 실행 기능을 다시 넣지 않는다.

## 변경 원칙

- 여러 파일을 바꾸는 작업은 PLANS.md에 현재 목표와 검증 상태를 남긴다.
- 사용자 변경과 legacy/를 보존한다.
- 파일 삭제, 원격 저장소 업로드, 외부 계정 변경, 유료 기능 사용은 현재 대화에서 명시적으로 허용된 경우에만 실행한다.
- 완료 보고에는 변경 파일, 검증 방법, 남은 위험을 짧게 적는다.

## 검증 원칙

- Markdown 내부 연결이 실제 파일을 가리키는지 확인한다.
- 현재 범위와 제외 범위가 문서마다 충돌하지 않는지 확인한다.
- 구현된 것과 설계된 것을 섞어 쓰지 않는다.
- 공식 Codex 기능을 설명할 때는 최신 OpenAI 공식 문서를 근거로 한다.
